A cleaner operations workspace for Bircan Migration. Review submissions faster, see real case risk, add internal notes, control final agent decisions, and run the core backend actions without digging through technical screens.

Simple admin, clearer decisions, less noise.

This version focuses on what matters in a real migration practice: what came in, what is risky, what action should be taken next, and what the final agent position should be. Technical detail is still available, but no longer drives the page.

Endpoint notes

Current frontend assumptions

This admin uses the same core backend pattern as your earlier page: /api/health, /api/meta/knowledgebase, /api/admin/submissions, /api/assessment/{id}/status, and /api/admin/submissions/{id}/resend-email. The Re-run AI button is prepared as a frontend placeholder unless your backend exposes a dedicated rerun endpoint.
